## Time Complexity

> 알고리즘의 성능을 나타내는 지표

입력 크기에 대한 연산횟수의 상한

### 🧐 알고리즘이 필요한 이유?

- 예시
  1부터 n까지의 수의 합을 구할 때?

  - 알고리즘 적용 x: O(n) - 단순 반복문

    ```
    for(int i = 1; i <= n; i++) {
      sum += i;
    }
    ```

  - 알고리즘 적용 o : O(1) - 가우스 공식
    ```
    sum = n * (n + 1) / 2;
    ```

### 가용범위

| 시간 복잡도 | 가용 범위                   |
| ----------- | --------------------------- |
| O(N!)       | 10P10 (순열, 조합 필요)     |
| O(2^N)      | 25, 26 (Back Tracking 필요) |
| O(N^2)      | 5천                         |
| O(NlogN)    | 300만 미만                  |
| O(N)        | 1억                         |
| O(logN)     | 10억                        |
